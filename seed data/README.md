# 🌱seed data🌱

👥 총 10명의 의뢰인, 6명의 변호사, 2개의 활성화된 openVidu 화상상담방을 가정하여 작업하였습니다.

📊 client 데이터는 기존에 소셜로그인 테스트를 하며 생긴 데이터들은 그대로 두고, 추가로 6명의 더미데이터를 만들었습니다.

📑 각 변호사의 프로필 사진 jpg/png 파일과 바이트배열 파일은 변호사 이름으로 되어있습니다. ex) 김태인 변호사 프로필 사진 파일이름: taein.jpg

🔐 변호사 계정들 비밀번호는 키보드를 영타로 둔채로 이름을 그대로 친 것 입니다!! ex) 송우석 변호사 비밀번호: thddntjr

⚠️ refresh 토큰은 백엔드의 토큰 생성 규칙에 맞춰 임의로 생성하여 DB에 삽입했습니다. <br>임의로 생성해놓은 것이다 보니 인증오류가 생길 가능성이 있어서 refresh 토큰이나 openVidu 토큰 관련 데이터는 참고로만 보시고 엔드투엔드 테스트 시에는 회원가입부터 진행하여 새로운 계정으로 테스트하는 것을 권장합니다.
